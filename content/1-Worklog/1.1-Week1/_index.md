## Overview
Week 1 focuses on AWS foundational concepts and account management. This week covers AWS Account setup, Budgets, Support plans, and IAM (Identity and Access Management) fundamentals.

## Tasks

### Day 2: AWS Account Setup & Exploration
- Create and verify AWS Account
- Configure AWS Console settings and preferences
- Setup Multi-Factor Authentication (MFA)
- Understand AWS Global Infrastructure and Services

### Day 3: Billing & Cost Management
- Create and configure AWS Budgets
- Setup Cost Allocation Tags
- Enable AWS Cost Explorer
- Monitor billing alerts and thresholds

### Day 4: AWS Support Plan & Documentation
- Review AWS Support Plans (Basic, Developer, Business, Enterprise)
- Select appropriate support tier
- Understand AWS Service Level Agreements (SLA)
- Explore AWS documentation and Knowledge Center

### Day 5: IAM Foundations
- Create IAM Users and Groups
- Configure IAM Policies and Permissions
- Understand Role-Based Access Control (RBAC)
- Create and manage access keys securely

### Day 6: IAM Advanced & Security Best Practices
- Implement password policies
- Setup MFA for IAM users
- Create and manage IAM Roles
- Review AWS Security Best Practices

## Achievements
✅ Successfully established AWS Account with Multi-Factor Authentication enabled  
✅ Configured AWS Budgets with cost threshold alerts  
✅ Implemented IAM user management with least-privilege access model  
✅ Created security group baseline policies following AWS Well-Architected Framework principles  
✅ Documented AWS account structure and organizational hierarchy  

## Resources Used
- AWS Management Console
- AWS IAM Best Practices Guide
- AWS Budgets Configuration
- AWS Documentation Portal

## Notes
- MFA is critical for account security - always enabled for root and privileged accounts
- IAM policies should follow the principle of least privilege
- Monitor billing regularly to avoid unexpected costs
